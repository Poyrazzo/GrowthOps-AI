# End-to-End Test & Bug-Fix Report
**Date:** 2026-06-12
**System:** GrowthOps AI — Growth Automation Platform
**Test campaign:** "Konuşarak Öğren — English Speaking Product"

---

## Executive Summary

The full outreach pipeline was tested end-to-end on **real infrastructure** (real website scrape, real OpenAI scoring + drafting, real Gmail SMTP/IMAP). After fixing **7 bugs**, the system now runs the complete flow:

**scrape → enrich → score → draft → human approval → send → delivery confirmed.**

A test email was generated by AI and **delivered to the real Gmail inbox**, verified via IMAP.

The single biggest issue (and the reason earlier tests appeared to "do nothing") was **not a code bug at all** — it was the café WiFi captive portal blocking all Docker container traffic. On a normal network everything works.

---

## Root Causes Found & Fixed

### 1. Café WiFi captive portal blocked all container traffic (environmental)
The FortiGate portal on the café network authorized the laptop but treated each Docker container as a separate unauthenticated device, intercepting HTTPS (fake certs) and blocking SMTP/IMAP ports.
**Symptom:** scrapes returned 132 bytes of portal redirect; OpenAI threw SSL errors; email failed with "Network unreachable."
**Fix:** switched to mobile hotspot. All four channels (scrape, OpenAI, SMTP, IMAP) immediately worked. *No code change needed — this is a network constraint, not a bug.*

### 2. Wrong email account silently won dispatch → email went to mock server
A leftover `test_sender@growthops.ai` account (GreenMail mock, daily-limit 100) from the built-in test suite out-competed the real Gmail account in `get_account_with_capacity()` ("most remaining capacity wins"). The message was marked `sent` but actually went to the throwaway mock server.
**Fix:** deactivated the mock account. Added this as a known footgun — mock/test accounts must be `is_active=False` in any real run.

### 3. Campaign activation never auto-scraped
The activation signal only fired on an explicit `update_fields` containing `status`, which the API PATCH never sets, so activating a campaign did nothing until the 6-hour beat tick.
**Fix:** rewrote `crm/signals.py` to track the previous status via `pre_save` and reliably detect any draft→active transition, then immediately enqueue scrapes for all non-LinkedIn sources.

### 4. "Restart Campaign" was a no-op-ish partial reset
It only re-queued drafting; it didn't clear stuck drafts, reset in-sequence leads, or clear scrape cooldowns, and it didn't reliably re-fire the activation signal.
**Fix:** `restart` now cancels open drafts + their approval entries, resets in-sequence leads to uncontacted, clears source cooldowns, and forces a clean draft→active transition that re-scrapes everything.

### 5. AI drafts contained unfilled `[Your Name]` / `[Your Position]` placeholders
The email prompt gave no rule against placeholders, so drafts shipped with bracketed gaps — not send-ready.
**Fix:** added strict prompt rules (no placeholders ever, sign off as "The <campaign> Team", <120 words, one CTA, plain text).

### 6. Generic role inboxes (info@, contact@) scored as junk / "unknown persona"
`info@englishtime.com` scored 30 with persona "unknown" because the scorer penalized the missing personal title — even though a contact inbox at a relevant school is a legitimate lead.
**Fix:** the scorer now receives the `is_generic_email` flag and judges role inboxes on **company fit** (label: "Company Gateway Inbox", ~55–75 when the company fits). The drafter writes a "please forward this to the right person" style note for generic inboxes instead of fake-personalizing.

### 7. UI dead-ends and missing visibility
"Add Source" button did nothing; leads couldn't be deleted; you couldn't tell which lead/source belonged to which campaign; "Queue for Outreach" was inert; dashboard showed fake hardcoded numbers.
**Fixes:**
- Wired "Add Source" to a real create-source modal (with campaign + type + priority).
- Added single + bulk lead deletion (API + UI), cleaning orphaned approvals.
- Added campaign / company / source attribution everywhere (leads list, lead drawer, sources cards).
- Wired "Queue for Outreach" to a real draft endpoint.
- Dashboard + campaign pages now show **real** data (no mock numbers).
- Added a live **Activity Monitor** page (auto-refresh every 4s) and a real per-campaign activity feed.

## Test-Mode Safety (added)

A global guard now guarantees that during testing **no real prospect is ever emailed**:
- `OUTREACH_TEST_MODE=true` redirects **every** outgoing email to `TEST_REDIRECT_EMAIL` (your own mailbox), regardless of the lead's address.
- The intended recipient is preserved in the `X-Intended-Recipient` header, the subject prefix `[TEST → real@addr]`, and the audit log.
- Set `OUTREACH_TEST_MODE=false` only when ready to email real prospects.

## Remaining Notes / Not Bugs

- Some seeded source URLs may 404 or block scrapers; a failed scrape simply yields 0 leads and is logged — harmless. Prune via the Sources page.
- Generic inboxes intentionally score lower than named decision-makers; raise/lower `LEAD_SCORE_THRESHOLD` to taste.
- Reply-loop classification path is wired and unit-tested by the built-in suite; not re-run live here because it needs an inbound reply to arrive.
